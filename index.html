📊 Resumen práctico de fórmulas DAX – Power BI

Chuleta para todos: qué datos necesitás + ejemplos copiables.

🔎

Tip: podés buscar por función (SUMX), por concepto (rolling), o por palabra dentro del DAX.

Incluye Time Intelligence Incluye contexto y filtros Ejemplos genéricos

⚠️ Para Time Intelligence necesitás una tabla Calendario marcada como Date table y relacionada por fecha con la tabla de hechos.

✅ Convenciones usadas en los ejemplos

  • Tabla (hechos): Ventas
  • Columnas: Ventas[Importe], Ventas[Cantidad], Ventas[Precio], Ventas[Cliente], Ventas[Fecha]
  • Calendario: Calendario[Fecha] (tipo Date)

1️⃣ Agregaciones básicas

🔹 SUM (suma)

Necesitás: 1 columna numérica → Tabla[ColumnaNumérica]
Ventas Totales = SUM ( Ventas[Importe] )

🔹 AVERAGE (promedio)

Necesitás: 1 columna numérica
Precio Promedio = AVERAGE ( Ventas[Precio] )

🔹 MIN / MAX

Necesitás: 1 columna (numérica o fecha)
Fecha Máxima = MAX ( Ventas[Fecha] )
Fecha Mínima  = MIN ( Ventas[Fecha] )

🔹 COUNT / COUNTA / DISTINCTCOUNT

Necesitás: 1 columna
  • COUNT → números
  • COUNTA → no vacíos (texto/números)
  • DISTINCTCOUNT → únicos
Cantidad Registros (num) = COUNT ( Ventas[Cantidad] )
Cantidad Registros (no vacío) = COUNTA ( Ventas[Cliente] )
Clientes Únicos = DISTINCTCOUNT ( Ventas[Cliente] )

2️⃣ IF / SWITCH

🔹 IF

Necesitás: condición lógica + resultado verdadero/falso
Resultado = IF ( [Ventas Totales] > 100000, "Alto", "Bajo" )

🔹 SWITCH(TRUE()) (recomendado)

Necesitás: varias condiciones
Categoria =
SWITCH (
    TRUE(),
    [Ventas Totales] > 200000, "Muy Alto",
    [Ventas Totales] > 100000, "Alto",
    "Normal"
)

3️⃣ CALCULATE (cambio de contexto)

🔹 Filtrar por una condición

Necesitás: una medida base + una columna para filtrar
Ventas 2025 =
CALCULATE ( [Ventas Totales], Calendario[Año] = 2025 )

🔹 Múltiples filtros

Ventas USA 2025 =
CALCULATE ( [Ventas Totales], Calendario[Año] = 2025, Ventas[Pais] = "USA" )

🔹 Quitar filtros

Necesitás: saber qué tabla/columna querés ignorar
Ventas Sin Filtro Pais =
CALCULATE ( [Ventas Totales], REMOVEFILTERS ( Ventas[Pais] ) )

4️⃣ BLANKs y divisiones

🔹 DIVIDE (evita errores)

Necesitás: numerador y denominador (medidas o expresiones agregadas)
Margen % = DIVIDE ( [Margen], [Ventas Totales], 0 )

🔹 COALESCE (reemplazar BLANK)

Necesitás: una medida/expresión que a veces devuelva BLANK
Ventas (0 si blank) = COALESCE ( [Ventas Totales], 0 )

5️⃣ Filtros por selección

🔹 SELECTEDVALUE

Necesitás: una columna de dimensión (ej. País, Cliente, Producto)
Elemento Seleccionado = SELECTEDVALUE ( Ventas[Pais], "Varios" )

6️⃣ Iteradores (funciones X)

🔹 SUMX (sumar una expresión fila a fila)

Necesitás: una tabla + una expresión
Importe Calculado =
SUMX ( Ventas, Ventas[Cantidad] * Ventas[Precio] )

🔹 AVERAGEX (promedio de una expresión)

Precio Prom (calc) =
AVERAGEX ( Ventas, Ventas[Importe] / Ventas[Cantidad] )

7️⃣ Time Intelligence (fechas)

🔹 YTD / MTD / QTD

Necesitás: medida base + Calendario[Fecha]
Ventas YTD = TOTALYTD ( [Ventas Totales], Calendario[Fecha] )
Ventas MTD = TOTALMTD ( [Ventas Totales], Calendario[Fecha] )
Ventas QTD = TOTALQTD ( [Ventas Totales], Calendario[Fecha] )

🔹 Últimos 3 meses (rolling)

Ventas Últimos 3M =
CALCULATE (
    [Ventas Totales],
    DATESINPERIOD ( Calendario[Fecha], MAX ( Calendario[Fecha] ), -3, MONTH )
)

🔹 Últimos 3 meses (sin incluir el mes actual)

Ventas 3M sin Actual =
CALCULATE (
    [Ventas Totales],
    DATESBETWEEN (
        Calendario[Fecha],
        EOMONTH ( MAX ( Calendario[Fecha] ), -4 ) + 1,
        EOMONTH ( MAX ( Calendario[Fecha] ), -1 )
    )
)

8️⃣ Ranking / Top N

🔹 RANKX

Necesitás: una dimensión + una medida para rankear
Rank Cliente =
RANKX ( ALL ( Ventas[Cliente] ), [Ventas Totales], , DESC )

9️⃣ Tips rápidos

  • Preferí medidas (no columnas) cuando sea posible.
  • Usá DIVIDE para evitar errores.
  • Para muchas condiciones: SWITCH(TRUE()).
  • Probá medidas en una tabla antes de llevarlas a gráficos.
